📋 STEP 1: TEST LOCALLY (2 minutes)
────────────────────────────────────────────────────────────────
Open terminal in this folder and run:

    npm install
    npm run dev

Then open: http://localhost:3000

✅ Test the full flow: Home → Form → Quiz → Results → Digital ID


📦 STEP 2: PUSH TO GITHUB (5 minutes)
────────────────────────────────────────────────────────────────
1. Create new repository on GitHub.com
   Name: personality-stamp

2. Run these commands in terminal:

    git init
    git add .
    git commit -m "Initial commit: Personality Stamp app"
    git remote add origin https://github.com/YOUR_USERNAME/personality-stamp.git
    git branch -M main
    git push -u origin main

   (Replace YOUR_USERNAME with your GitHub username)


🚀 STEP 3: DEPLOY TO VERCEL (5 minutes)
────────────────────────────────────────────────────────────────
1. Go to: https://vercel.com
2. Sign in with GitHub
3. Click "New Project"
4. Import your "personality-stamp" repository
5. Click "Deploy" (use default settings)
6. Wait 2-3 minutes
7. ✅ Done! Your app is live!


🗄️ STEP 4: SUPABASE (OPTIONAL - 10 minutes)
────────────────────────────────────────────────────────────────
Only if you want database persistence:

1. Go to: https://supabase.com
2. Create new project
3. Run SQL from "supabase-setup.sql" (in SQL Editor)
4. Create "selfies" bucket in Storage (make it PUBLIC)
5. Get URL and Key from Settings → API
6. Add to Vercel:
   - Go to Vercel → Your Project → Settings → Environment Variables
   - Add: NEXT_PUBLIC_SUPABASE_URL
   - Add: NEXT_PUBLIC_SUPABASE_ANON_KEY
7. Redeploy your project

See "SUPABASE_SETUP_GUIDE.md" for detailed instructions.

❓ TROUBLESHOOTING
────────────────────────────────────────────────────────────────
Camera not working?
  → Needs HTTPS (Vercel provides this automatically)
  → Check browser permissions (click lock icon)

Build fails?
  → Run "npm run build" locally first
  → Check error message in Vercel logs

Supabase not connecting?
  → App works without it (uses localStorage)
  → Verify environment variables in Vercel
  → Restart dev server after adding .env.local
